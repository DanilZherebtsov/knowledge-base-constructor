## Заполнение слотов (по карте template-anatomy)

- **S1** — слово контекста «дело»; «О проекте» = что за бизнес/проект + ограничения (бюджет, юр.форма/налоговый режим, ценности/табу, тайминг, ключевые люди).
- **S2** — type-папка `entities/` (контрагенты и люди) добавлена к базовой четвёрке. Группировка префиксами: `supplier- client- contractor- staff- landlord- partner-`.
- **S3** — формат `entities/` (карточка контрагента: Status/Кто/Контакты/Условия + ## История + ## Связи); префиксы для `discovery/`: `market- competitor- norm- price-`. Заполнено в page-conventions.md.
- **S4** — пусто (рабочий слой = только `output/` с подпапкой `черновики/`).
- **S5** — 7 фиксированных разделов STATE (см. state-sections выше) + развёрнутая механика «Календарь обязательств» (раздел 5) в state-rules.md.
- **S6** — `decision-lifecycle.md` (вопрос в STATE → сравнение в output/черновики/ → ADR в decisions/ → каскад в entities/).
- **S7** — «Источники побеждают вики» (класс без кода).
- **S8** — валюта и формат сумм из bootstrap-интервью (дефолта нет).
- **RAW-DEFAULTS** — динамические подпапки raw/ (bootstrap заводит под названные источники + прочее/).
- **INTERVIEW-Q** — 3 вопроса (см. interview выше).
- **KNOWLEDGE-UNIT** — страница (base-дефолт, не `claim`); НЕ заполнялось — наследуется из base.
- **DOMAIN-LINT** — проверка «Календаря обязательств» STATE (близкие/просроченные).
- **CLOSE-OP** — `decision-closed | <выбор>` в логе.
- **ROLES-FILL** — примеры ролей: юрист, финансист, снабженец, маркетолог; `roles/_шаблон.md` дозаполнен конкретным примером роли «юрист».

## Механики

- **roles** — **не механика класса, а базовая универсальная машинерия** (`methodology/roles.md`, `roles/_шаблон.md`, раздел «Роли» в CLAUDE.md — всегда из base, ADR-0027). Пресет лишь заполняет `ROLES-FILL` примерами (юрист/финансист/…). Прежнее «business подключил роли как опц. механику» (ADR-0004/0015) снято ADR-0027 — роли теперь у всех.
- **decision-lifecycle** — есть `methodology/decision-lifecycle.md` (слот S6), отсылки из CLAUDE.md «Поток решения» и из «Дисциплины» п.8.
- НЕ используются: claim-graph (research), spec-lifecycle (saas), question-lifecycle (research).
