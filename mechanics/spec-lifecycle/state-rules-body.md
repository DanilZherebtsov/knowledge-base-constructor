# Тело state-rules для класса с кодом (spec-lifecycle)

Вставляется wiring'ом в base `state-rules.md`: замещает регион слота S5 («## Структура») и уточняет правило 2б. Базовые правила 1, 3–7 — из base.

## Структура (7 разделов)

1. **Стадия** — где мы в lifecycle и какова текущая цель (идея / прототип / MVP / бета / v1 / v2).
2. **Путь до цели** — milestones от текущего момента до цели, по порядку. Весь путь в одном месте.
3. **Сейчас в работе** — что в флайте прямо сейчас (обычно одна-две задачи). Файл спеки — в `specs/` (`<feature>-NNN-...`, `status: active`). Спринтом — ссылка на активную спринт-спеку (`SPRINT-<NAME>.md`) + активную задачу; **список задач спринта в STATE не дублируется** (он в спринт-спеке).
4. **Следующее (1–2 недели)** — бэклог; одностроки здесь, файл спеки заводится при взятии в работу.
5. **Завершено за последнюю неделю** — буферная зона перед схлопыванием.
6. **Блокеры и риски**.
7. **Известный техдолг** — список ссылок на ADR со статусом `active` и тегом `tech-debt`.

**Гранулярность отражает горизонт понимания.** В начале «Путь до цели» — 2–3 строки крупными мазками («сформулировать концепцию», «10 интервью», «собрать MVP»). По мере приближения каждый пункт дробится. Не атомизировать преждевременно — это ложная точность.

## Правило 2б (перенос единицы работы)

После завершения значимого пункта Claude переносит его из «Сейчас в работе» в «Завершено за последнюю неделю».
