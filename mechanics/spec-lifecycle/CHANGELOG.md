# CHANGELOG — механика spec-lifecycle

Версия — в [versions.json](../../versions.json) (`mechanics.spec-lifecycle`). Формат и дисциплина записей — как в [base/CHANGELOG.md](../../base/CHANGELOG.md): каждый bump несёт запись `## vN` с меткой влияния и конкретной сутью; привратник публикации сверяет номер с верхней записью.

## v2
`[инструкция]` Между спекой и `completed` — обязательный **цикл исполнения**: спека раньше кода → гейт из 4 независимых субагентов (3 на корректность + 1 на влияние на смежные модули) → реализация → повторный гейт → доклад; петля на ошибках; пробел в скоупе уходит обратно в спеку, блокер вне скоупа — к человеку с вариантами. Always-on-указатель вставляется wiring'ом механики в «Принципы → фаза действуй» (не правкой base — base делят классы без кода). Знание зафиксировано в ADR-0016. Знания вики не трогаются.

## v1
Учредительная версия. Единица работы для классов с кодом: спека/спринт в `specs/` со статусом во frontmatter, доменный поток (бэклог/вопрос → файл в `specs/` со `status: active` → извлечение ADR при acceptance → синтез знаний по всем типам вики). Подробности механики и wiring — в [_about.md](_about.md).
