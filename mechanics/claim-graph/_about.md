# Механика: claim-graph

**Что делает (1 строка):** заменяет «единицу знания = страница» на «единицу знания = claim» и добавляет явный граф эпистемических связей (`supports/contradicts/refines`) между атомарными утверждениями с обязательной локализацией каждой цитаты.

## Слоты целевого проекта, которые трогает

- **S2** — набор type-папок `wiki/`: добавляет `claims/` (как центральный тип; для research — вместо `discovery/`).
- **S3** — центральный/доменный тип + его формат: claim-evidence chain.
- **S7** — правило авторитета: «источники побеждают вики» + добавка «каждая цитата с локализацией, lint-проверяемо».
- **KNOWLEDGE-UNIT** (в `ingest.md`) — значение `claim`.
- **DOMAIN-LINT** (в `lint.md`) — баланс counter-evidence у active/validated claim'ов.

## Файлы целевого проекта, которые трогает

- `methodology/page-conventions.md` — frontmatter (поля claim), формат типа `claims/`, правило цитаты, локализация.
- `methodology/ingest.md` — слот KNOWLEDGE-UNIT, шаг «извлечь claim-кандидаты», 4-веточная компиляция, каскад по `claims/`+`synthesis/`, граница claim/decision.
- `methodology/lint.md` — слот DOMAIN-LINT, claim-автоисправления, report-проверка цитат без локализации.
- `methodology/query.md` — особые формы query (обратный поиск по `sources:`, evidence-карта, инвентарь противоречий) + различение уверенности по claim-статусу/Evidence/Counter-evidence.
- `methodology/index-log-format.md` — claim-строка индекса: одно-предложная формулировка claim'а + статус в скобках.
- `CLAUDE.md` — дерево `wiki/` (объявление типа `claims/`), описание типа в «Wiki: типы страниц», слот S7.

## Зависит от механик

- Ничего жёсткого: claim-graph additive поверх голого base. Несовместима только с другой механикой, претендующей на **тот же** центральный тип (architecture / entities) — claim'ы и architecture не уживаются как один центральный тип в одном проекте; выбрать один.

## Пошаговый wiring

1. **Положить файлы механики.** `mechanics/claim-graph/claim-graph.md` (вся механика) + этот `_about.md` — это исходник для конструктора, в целевой проект не копируются как есть; их содержимое вплетается по шагам ниже.

2. **`CLAUDE.md` → слот S2 (дерево `wiki/`).** В блоке дерева добавить строку типа `claims/` и (для research) убрать `discovery/`:
   ```
   claims/        (атомарные утверждения с цепочкой evidence; центральный тип.
                   Группировка через префиксы: topic-, hypothesis-, finding-, definition-)
   ```

3. **`CLAUDE.md` → раздел «Wiki: типы страниц и операции».** Добавить буллет про `claims/` (что это центральный тип, одна страница = утверждение + evidence chain + связи `supports/contradicts/refines`).

4. **`CLAUDE.md` → слот S7 (правило авторитета).** Заполнить: «**Источники побеждают вики**» + «**каждая цитата с локализацией**, lint-проверяемо».

5. **`page-conventions.md` → frontmatter.** В type-перечень добавить `claim`; добавить опциональные поля `supports / contradicts / refines` с пометкой «только для claim»; в статусы добавить `open/validated/invalidated`. Вставить абзац «Связи `supports / contradicts / refines`» (Кусок 1 механики).

6. **`page-conventions.md` → слот S3 (формат типа).** Вставить блок «### claims/ — атомарные утверждения» с префиксами имён и полным телом claim-страницы (Несущий формат механики), включая «Правило цитаты» и «Когда заводить новый claim».

7. **`page-conventions.md` → секция атомарности + локализация цитат.** Добавить claim-лимит ≤ 100 строк (разбивать через `refines`). Само правило «**Цитата — с локализацией**» с base@26 стоит в base безусловно — **не дублировать его**; добавить только доменные форматы локализации и ужесточение: пункт Evidence/Counter-evidence без локализации считается недокументированным и помечается lint'ом (Кусок 4).

8. **`ingest.md` → слот KNOWLEDGE-UNIT.** Заполнить значением `claim`. Вставить шаг «Извлечь claim-кандидаты» перед компиляцией; заменить generic-ветки шага «скомпилировать» на 4 ветки (тот же / refines / contradicts / новая концепция); в «Каскад» указать сканирование `wiki/claims/` и `wiki/synthesis/`; добавить «Особый случай — полевой материал» (Кусок 2). Добавить шаг **«Методологические тезисы — в `decisions/`»**: если источник несёт методологический выбор — это ADR, а не claim (граница: claim = «X есть Y»; decision = «рассматриваем X через Y»). В шаге «Разместить сырой файл» заменить generic «удалить секреты» на research-этику: «Удалить личные данные респондентов до сохранения, если этического согласия на их публикацию нет».

9. **`lint.md` → слот DOMAIN-LINT.** Заполнить проверкой «claim active/validated, у которого Counter-evidence перевешивают Evidence — кандидат на ревизию». В «Автоисправление» добавить починку битых claim-связей и взаимных ссылок. В «Только отчёт» добавить: claim'ы > 100 строк; устаревшие противоречащие claim'ы; **цитаты без локализации** в формате `<страница>:<номер строки> — <фрагмент>` (Куски 3 и 4).

10. **`query.md` → особые формы.** Добавить раздел «Особые формы query»: обратный поиск по `sources:` (что опирается на источник), evidence-карта вокруг claim'а, инвентарь противоречий (рёбра `contradicts`). В шаге уровня уверенности — различать по claim-статусу + балансу Evidence/Counter-evidence.

11. **`index-log-format.md` → claim-строка.** Уточнить: для claim'ов резюме в индекс-строке = сама формулировка claim'а в одно предложение + статус в скобках, напр. `- [claim-...] — <формулировка> _(validated)_`.

12. **Чего НЕ делать.** Не дублировать в целевой проект generic-каркас base (атомарность 400/800, cross-links, journal-страницы, generic ingest-шаги размещения raw / index / log / commit, generic lint авто/report-уровни, template-update-гейт, форматы decisions/synthesis/principles/discovery) — он уже в base. Из base убирается только дефолтный `discovery/` (его место занимает `claims/`); остальные generic-типы остаются.
