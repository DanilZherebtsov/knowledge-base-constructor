# Лента миграций конструктора

**Структурные** обновления конструктора (раскол/слияние/переименование механик, сдвиг архитектуры) профилактика не может вывести подменой файла — для них здесь лежат **упорядоченные пошаговые инструкции**. Рутинные обновления (`[инструкция]`/`[формат]`) идут как раньше, без файла миграции. Модель — ADR-0024 (в приватном мета-репо конструктора).

**Формат файла** — `NNNN-<slug>.md`, frontmatter:

| Поле | Смысл |
|---|---|
| `id` | Монотонный номер (порядок в ленте). |
| `requires` | Входное состояние: версии частей отпечатка, при которых миграция применима. |
| `produces` | Выходное состояние: версии частей после применения. |
| `impact: структурная` | Метка `[миграция]` — не file-swap. |
| `applies-to` | Пресет(ы), к которым относится. |

Тело — агент-исполнимые шаги против живого боевого репо + **встроенный артефакт-гейт** (diff обязан показать только СТАНДАРТ-слой; код/данные/тела вики не тронуты).

**Как применяется (профилактика).** Отставший проект проигрывает общую ленту от своей позиции до HEAD **строго по порядку**: рутинный шаг — подменой файла / миграцией заметок по согласию; структурный — исполнением его инструкции. Не переупорядочивать, не пропускать. Позиция выводится из пер-партовых версий отпечатка: проект «нуждается в миграции k», если его версии удовлетворяют `requires`, но не `produces`. Идемпотентно (применённая миграция больше не матчит `requires`).

**Дисциплина (для разработчика конструктора).** Сделал структурную правку → заведи здесь файл миграции в том же изменении (write-side гейт, близнец changelog-привратника). Без него downstream упрётся в разрыв.
