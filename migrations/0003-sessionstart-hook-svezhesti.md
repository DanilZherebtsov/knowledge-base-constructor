# Миграция 0003 — SessionStart-хук свежести (`.claude/`) + гигиена STATE в lint

> **Исполняется агентом ВНУТРИ боевого репозитория инстанса.** Подходит любому собранному проекту, чей отпечаток несёт `base@≤27`. Источник base-скелета — публичное зеркало `knowledge-base-constructor` (`constructor/base/`, тот же канал, что профилактика использует для сверки версий по SHA). Решение — ADR-0032 (хук) и ADR-0033 (проверки STATE).

## Что меняется и главный инвариант

`base@28` несёт два аддитивных улучшения:
1. **`.claude/`** — `settings.json` + `hooks/freshness_check.py`: `SessionStart`-хук считает возраст последней полной `lint` и `_Обновлено:_` STATE и вбрасывает отклонения (> 7 дней) в контекст на старте. Форсинг стартовых проверок, которые прозой протекают.
2. **Проза `base@28`** — в `CLAUDE.md` указание на хук (блок стартовых проверок + дисциплина №5 + строка `.claude/` в дереве); в `methodology/lint.md` — блок «Гигиена STATE.md» (report-only проверки размера/дрейфа); в `bootstrap.md` — `.claude/` в описи шаблона.

> **Инвариант миграции.** Меняется **только СТАНДАРТ-слой** — появляется `.claude/` (settings.json + hook), плюс каркас-проза `CLAUDE.md`, `methodology/lint.md`, `bootstrap.md`, отпечаток сборки. Всё остальное — байт-в-байт: `wiki/` (тела страниц), `raw/`, `STATE.md`, `output/`, `specs/`, `src/`, `data/`, любые уже существующие хуки и настройки в `.claude/`. Проверяется diff'ом (Шаг 5).

## Шаг 1. Определить входное состояние

Прочитать отпечаток (строка 3 `CLAUDE.md`) — убедиться `base@≤27`. Проверить `.claude/` на диске:
- **Ветка A — нет `.claude/settings.json`** (папки нет или в ней нет settings). → Шаг 2A.
- **Ветка B — `.claude/settings.json` УЖЕ есть** (у инстанса свои хуки/права). → Шаг 2B (слияние, не перезапись).

## Шаг 2A. Установить `.claude/` целиком (ветка A)

Из зеркала (`constructor/base/.claude/`):
1. `base/.claude/hooks/freshness_check.py` → `.claude/hooks/freshness_check.py` (создать папки; `chmod +x`).
2. `base/.claude/settings.json` → `.claude/settings.json` (копия как есть — слотов не несёт).

## Шаг 2B. Влить хук в существующий `.claude/settings.json` (ветка B) — СЛИЯНИЕ

**Не перезаписывать файл.** Скопировать `base/.claude/hooks/freshness_check.py` → `.claude/hooks/freshness_check.py`. В существующем `settings.json` — добавить объект хука в массив `hooks.SessionStart` (создать ключ, если его нет), **не трогая** прочие ключи (`permissions`, другие события, уже стоящие `SessionStart`-хуки):
```json
{ "matcher": "startup|resume|clear",
  "hooks": [ { "type": "command",
    "command": "python3 \"$CLAUDE_PROJECT_DIR/.claude/hooks/freshness_check.py\"" } ] }
```
Если такой хук уже стоит (повторный прогон миграции) — не дублировать. Идемпотентно.

## Шаг 3. Обновить прозу СТАНДАРТ-слоя по `base@28`

Из зеркала (`constructor/base/`), сохраняя instance-контент (заполненные слоты, «О проекте», доменные разделы):
1. `CLAUDE.md` — в блоке «Операционное состояние» добавить абзац «Форсинг этих проверок — хук `.claude/`…»; в дисциплине №5 — хвост про `SessionStart`-страховку; в дерево «Архитектура» — строку `.claude/`.
2. `methodology/lint.md` — в блок «## Только отчёт» добавить подраздел «### Гигиена `STATE.md`» (6 report-only проверок; пороги ~2000 симв / ~30 КБ / ~10 КБ / 7–14 дн).
3. `methodology/bootstrap.md` — в перечень «шаблон уже содержит …» добавить `.claude/`.

> Если инстанс несёт механику `spec-lifecycle`, профилактика отдельно подтянет `spec-lifecycle@6` (буллет «Дрейф статусов спек» в тот же блок «Только отчёт») — это рутинный `[инструкция]`-апдейт части, не входит в эту миграцию.

## Шаг 4. Обновить отпечаток сборки

Строка 3 `CLAUDE.md`: поднять `base@N` → `base@28`. Остальные части — без изменений.

## Шаг 5. Проверить

- `git diff` — тронуты только `.claude/`, `CLAUDE.md`, `methodology/lint.md`, `methodology/bootstrap.md`, отпечаток. `wiki/`, `raw/`, `STATE.md`, рабочие слои — нетронуты.
- Хук исполним: `CLAUDE_PROJECT_DIR="$PWD" python3 .claude/hooks/freshness_check.py` — при просроченной профилактике печатает JSON с `additionalContext`, при свежей — молчит, всегда `exit 0`.
- **Профилактика** после миграции: сравнение частей с апстримом показывает `base@28`; доступных обновлений этого перехода больше нет.
