# Конвенции wiki-страниц — frontmatter, типы, формат

Читать перед созданием страницы, правкой frontmatter, ревизией правил.

## Frontmatter

```yaml
---
type: <<СЛОТ S2: перечень типов класса — decision | discovery | synthesis | principles | ...>>
status: active | superseded | draft   # + доменные при необходимости (open/validated/invalidated, inactive…)
updated: YYYY-MM-DD
sources:                  # обязательно
  - ../../raw/<концерн>/2026-05-18-slug.md
  - https://...
supersedes: [adr-0007]    # опционально
superseded_by: [adr-0014] # опционально
tags: [тема1, тема2]
---
```

**`sources:` обязательно.** Утверждение без источника — галлюцинация под видом факта; lint не пропускает страницы без источников.

**Цитата — с локализацией.** Ссылка на источник — с конкретным местом (страница / параграф / таймкод); без него это пересказ, а не цитата. Формат места — доменный (см. слот S8). Правило записи: касается новых страниц, ретроактивная чистка старых не требуется.

## Атомарность

Одна концепция на страницу. Мягкий лимит 400 строк, жёсткий 800. Перевалило за 400 — выделить подстраницы и связать. Любое отдельное чтение остаётся в пределах контекстного окна.

## Перекрёстные ссылки

Markdown-ссылки с относительными путями: `[Название](../type/slug.md)` для wiki, `[Описание](../../raw/.../file.md)` для сырых, `[Название](https://...)` для URL. Анкор-текст обязателен и человекочитаем.

## Страницы-журналы

Накапливающая наблюдения страница: каждая запись начинается с `## [YYYY-MM-DD] Заголовок` — видно, что свежо.

## Префиксы имён файлов

Внутри плоской type-папки страницы группируются **префиксом имени** — ведущим токеном файла (`<префикс>-<slug>.md`). Префикс заменяет подпапки (глубина = 1) и служит ключом фильтрации: виден в ссылках `index.md` и через `ls <type>/<префикс>-*`. Так роль/query находит срез, не вчитывая всю вику.

## Типы страниц — формат

### decisions/ — ADR (универсально)

```
# ADR-0012 — <решение одной фразой>

Status: active
Date: 2026-05-18
Context: <проблема и ограничения>
Decision: <что решили>
Consequences: <что проще, что сложнее>
Alternatives considered: <кратко, с обоснованием отказа>
Sources: <цитаты/ссылки>
```
Решения помечаются `superseded`, не удаляются. Поток до ADR — <<СЛОТ S6: lifecycle-файл класса>>.

### synthesis/ — записанные обратно ответы

Сравнения, ретроспективы, сквозные анализы, брифинги. Обязательны: введение (что за вопрос) и ссылки на источники/страницы, на которых построено. Synthesis переписываем (правим существующий, а не плодим новый).

### principles/ — правила из работы

Правило «всегда X / никогда Y» из инцидента → страница по applicability (`process.md`, `domain.md`, …). Формат записи: `## P-XX — имя`, **Правило / Почему / Когда применять / Прецедент / Источник**. Без источника принцип не записывается. Читаются перед нетривиальной задачей (фаза «подумай»).

### discovery/ — знание о внешнем мире

Сегмент, конкурент, факт, норматив, инсайт. Группировка префиксами. У гипотез — статус `open`/`validated`/`invalidated`.
