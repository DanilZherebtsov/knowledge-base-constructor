# Bootstrap — инициализация под конкретный проект

Запускается при первом ingest в проект без структуры или по просьбе («инициализируй структуру», «разверни вики»). **Никогда** из `query`/`lint`. Один раз; повторно — только восстановление.

Bootstrap — не штамповка фиксированных папок, а **короткая настройка под проект**. Шаблон уже содержит `CLAUDE.md`, `STATE.md`, `methodology/`, `roles/`; bootstrap достраивает рабочие слои.

## Шаг 1 — короткое интервью (3–5 вопросов, дождаться ответов)

<<СЛОТ INTERVIEW-Q: вопросы интервью класса (дефолт ниже)>>

- Что за проект и в чём сейчас цель?
- Какие источники будут приходить?
- Доменные конвенции (единицы, формат цитат — слот S8), если есть.

**Всегда (любой класс, вне слота выше) — спросить про валюту:** «В какой валюте деньги проекта и в каком формате писать суммы?» **Дефолта нет** — не предполагать валюту по языку или стране; ответ фиксируется в слоте S8 «Документы и нейминг» `CLAUDE.md` (пример формата — с разделителем тысяч: `220 000 ₽`, `$1,200`, `€990`). Проект без денежных сумм — так и записать («суммы не используются»). Пропустить вопрос, если S8 уже заполнен.

**Всегда (любой класс, вне слота выше) — спросить про код:** «Будет ли в проекте собственный код — сайт, лендинг, бот, приложение, скрипты?»

- **Да** → подключить работу с кодом (механика `software-engineering`: код-папка в корне, цикл исполнения, дисциплина владения, для веб-продукта — пара ролей «правит / деплоит»).
- **Нет** → ничего не подключаем; способность можно **создать позже в любой момент** — фразой «будем писать код» или автоматически, когда код впервые придёт (привратник в [ingest.md](ingest.md) предложит сам).
- Пропустить вопрос, если механика `software-engineering` уже подключена.

## Шаг 2 — предложить структуру

На основе ответов Claude **предлагает** раскладку (не создаёт молча): <<СЛОТ RAW-DEFAULTS: стандартные подпапки `raw/` класса; дефолт — под названные источники + `прочее/`>>; особые папки, если всплыли. Человек подтверждает/правит.

## Шаг 3 — создать (чего нет, не перезаписывая существующее)

- `input/.gitkeep` — свалка для входящих (человек кидает сюда любые материалы; Claude раскладывает по `raw/` на ingest, человеку подпапки знать не нужно).
- `raw/.gitkeep` + согласованные подпапки.
- `wiki/` + type-папки класса (<<СЛОТ S2>>, каждая с `.gitkeep`); `wiki/index.md` (`# Wiki Index`); `wiki/log.md` (`# Wiki Log` + `## [YYYY-MM-DD] bootstrap | инициализация`).
- `output/.gitkeep` — **корень рабочих файлов, есть у всех классов**; создаётся пустым, подпапки (`drafts/`, `черновики/`, папки под живые документы) — по факту, не заранее. <<СЛОТ S4: дополнительные слои для классов с кодом — `specs/` + `src/` + `data/`>>

## Шаг 4 — заполнить сопутствующие файлы

- «О проекте» в `CLAUDE.md` из ответов интервью. После заполнения **удалить инструктивную сноску** — блок `>` под заголовком «О проекте» (он для составителя, не для проекта).
- **Дерево «Архитектура» в `CLAUDE.md`** — вписать созданные подпапки `raw/` (в шаблоне их нет).
- Стартовый раздел `STATE.md` (профиль/стадия); `_Обновлено:_` = сегодня.

## Что НЕ создаётся при bootstrap

- **Роли** — заранее не заводим; сообщить, что любую роль можно создать в любой момент «создай роль <название>» (см. [roles.md](roles.md)), и что по желанию можно заранее закинуть в `input/` описание роли и материалы для её работы — тогда она создастся сразу квалифицированно.
- **Код-папка** — пустой не заводится. Подключили работу с кодом (вопрос про код в Шаге 1) — папка появляется, когда код реально приходит или создаётся, по решению человека. Не подключили — можно добавить способность позже в любой момент.
- Папки сверх минимума, **включая подпапки рабочего слоя** (`drafts/`, `черновики/`), — по смыслу, когда появится потребность (правило «Структура растёт по мере надобности» в [CLAUDE.md](../CLAUDE.md)).

После bootstrap — если под git — коммит `bootstrap: инициализация структуры`; без git пропускается.
