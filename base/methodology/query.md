# Query — структурированный запрос с цитатами

Запускается фразами: «что мы знаем про X», «сделай справку по Y», «сравни A и B по вики», «обоснуй решение Z», «мы уже это обсуждали?».

1. Прочитать `wiki/index.md` и отобрать страницы-кандидаты — фильтруя по **префиксу имени** и, при необходимости, `grep` по полю `tags:` во frontmatter. В контекст идут только релевантные страницы, не вся вика.
2. Прочитать эти страницы (на шаг по ссылкам, если нужно). На «что мы знаем про…» — **сначала вики, потом сырые источники**, не по памяти разговора.
3. Синтезировать ответ с цитатами в формате markdown-ссылок: `[Название](wiki/<type>/<slug>.md)` для wiki, `[Описание](raw/<концерн>/<file>.md)` для сырых, `[Название](https://...)` для URL. Уровень уверенности читается из структуры (число и независимость источников, связанные решения, наличие контр-аргументов), не из числовых оценок — ответ это отражает.
4. Вскрылся пробел — предложить шаг дальше: новую страницу, недостающий источник, устаревшее утверждение на перепроверку.
5. Содержательный ответ (сравнение, ретроспектива, обоснование, брифинг) — предложить запись-обратно в `wiki/synthesis/`. **Записанные ответы накапливаются; ответы в чате — испаряются.**

Обычные запросы не пишут в файлы. Только запись-обратно (после явного «да») и явный ingest меняют вики.
