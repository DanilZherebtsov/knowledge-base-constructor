# Формат `index.md` и `log.md`

## index.md

Группируется по типу (<<СЛОТ S2: секции = набор типов класса>>), внутри типа — по алфавиту. Каждая строка:

```
- [Название страницы](<type>/<slug>.md) — однострочное резюме. _Updated: YYYY-MM-DD_
```

Ссылка — **относительно самого `index.md`** (лежит в `wiki/`): префикс `wiki/` не нужен. То же для ссылок между страницами: путь относителен от текущей (со `wiki/decisions/foo.md` на `wiki/discovery/bar.md` — это `../discovery/bar.md`).

## log.md

Только дополнение, новые записи внизу. Префикс `## [YYYY-MM-DD] <op> | <subject>` — для grep (`grep "^## \[" wiki/log.md | tail -20`).

Дата записи — дата окружения на момент записи, не mtime файла и не дата, когда делалась сама работа. Из режима «только дополнение» следует инвариант: дата каждой новой записи ≥ даты предыдущей. Свежая запись с более ранней датой почти всегда значит, что дата взята с mtime, — это нарушение, которое ловит lint.

Операции в логе:
- `ingest | <источник>` — после каждого ingest, с перечнем каскадно-обновлённых страниц.
- `lint | найдено N проблем, исправлено M` — после каждого lint.
- `bootstrap | инициализация` — единожды, при первом разворачивании структуры.
- <<СЛОТ CLOSE-OP: доменная операция закрытия единицы работы — research: `question-closed | Q-NNN`; business: `decision-closed | <выбор>`; saas: нет>>.
