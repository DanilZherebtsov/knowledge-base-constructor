## v16
`[инструкция]` Flow B: достроен канал чтения, который v15 оставил незакрытым. (1) Когда шелл без сети (песочница) — читать теми же двумя шагами через `WebFetch` (живой GET мимо локальной песочницы), не сдаваться в веб-поиск. (2) **`WebSearch` для этой проверки запрещён** — поисковый индекс отдаёт снимок многодневной давности; именно он подсунул фантомный `base:12` на свежем апстриме (рецидив после v15). (3) Число «апстрим» засчитывается только из SHA-верифицированного живого чтения; не вышло ни шеллом, ни `WebFetch` — проверка НЕ выполнена, число из кэша/поиска/памяти в таблицу не ставится, «совпадает» не выводится. Знания не трогаются.

## v15
`[инструкция]` Flow B: проверку версий читать **по фиксации коммита, а не по ветке `main`** — `git ls-remote` за актуальным HEAD, затем raw-URL с этим SHA в пути. Отменяет ошибочную посылку v11, будто contents API «свежий без кэша»: и contents API, и `raw/main` — независимые пограничные кэши, каждый может отстать, поэтому два чтения одной ветки дают расходящиеся числа без способа выбрать верное (реальный кейс: API отдал 12, raw — 8 при истинных 14). URL с SHA адресуется содержимым — кэш не вернёт чужие байты, расхождение «какому чтению верить» исчезает по построению. Знания не трогаются.

## v14
`[инструкция]` Flow B сделан БЕЗУСЛОВНЫМ: проверка версий идёт на каждом lint независимо от состояния вики (пустая/свежая/неизменная/N-й прогон — не повод пропустить) и НЕ завершается без двух показанных артефактов — вставленного тела `versions.json` и таблицы сравнения по частям с числами; «обновлений нет» допустимо, только если в таблице `у меня == апстрим` у каждой части. Закрывает рецидив: на свежеразвёрнутой пустой вике lint трижды отрапортовал «совпадает», не сходив за живым `versions.json` (base@10/@11 хардили текст, но контекст «вика пустая» пересиливал). Знания не трогаются.
