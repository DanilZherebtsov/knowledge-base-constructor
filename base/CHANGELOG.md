# CHANGELOG — base (скелет)

Версия `base` — в [versions.json](../versions.json). Растёт при правке скелета или универсальной methodology; собранные проекты ловят это на профилактике по отпечатку (часть `base@N`).

**Дисциплина записей (для разработчика конструктора).** Каждая версионируемая часть несёт собственный changelog: base — этот файл, механика — `mechanics/<имя>/CHANGELOG.md`. Правило жёсткое: **поднял версию части в `versions.json` — в ТОТ ЖЕ коммит впиши запись `## vN`.** Без неё downstream-проект ловит на профилактике бамп, который не может ни классифицировать, ни объяснить человеку (реальные кейсы: base@18 и spec-lifecycle@2 уехали без записей — профилактика верно отказалась применять их вслепую). Привратник в `publish-constructor.yml` блокирует публикацию зеркала, если номер версии части и верхняя запись её changelog разошлись.

**Что в записи (иначе она бесполезна для триажа на той стороне):** метка влияния — `[инструкция]` (проза/правило, знания вики не трогаются) или `[формат]` (новое поле/раздел на страницах вики → нужна миграция, дописать строку «как принять»: что мигрировать, lazy/eager/skip); и **конкретная суть** — какие файлы/правила изменены и зачем, в терминах, по которым downstream сам классифицирует радиус и переведёт человеку. Не «выровнено по vN»: скупая запись прячет правки.

Закрывает кейс боевого инстанса: готовый лендинг не распознался как живой код — ingest унёс сайт в `raw/`, агент предложил вести его в `output/`. Знания не трогаются.

## v15
`[инструкция]` Flow B: проверку версий читать **по фиксации коммита, а не по ветке `main`** — `git ls-remote` за актуальным HEAD, затем raw-URL с этим SHA в пути. Отменяет ошибочную посылку v11, будто contents API «свежий без кэша»: и contents API, и `raw/main` — независимые пограничные кэши, каждый может отстать, поэтому два чтения одной ветки дают расходящиеся числа без способа выбрать верное (реальный кейс: API отдал 12, raw — 8 при истинных 14). URL с SHA адресуется содержимым — кэш не вернёт чужие байты, расхождение «какому чтению верить» исчезает по построению. Знания не трогаются.
