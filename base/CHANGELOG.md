# CHANGELOG — base (скелет)

Версия `base` — в [versions.json](../versions.json). Растёт при правке скелета или универсальной methodology; собранные проекты ловят это на lint по отпечатку (часть `base@N`).

## v9
`[инструкция]` Папка `input/` — свалка для входящих: человек кидает любые материалы как есть, Claude на ingest сам раскладывает их по `raw/` (тип + имя) и компилирует в `wiki/`; подпапки `raw/` человеку знать не нужно. Правки: дерево «Архитектура» `CLAUDE.md`, `ingest.md` (шаги 1–2), `bootstrap.md` (создаётся `input/.gitkeep`). Знания не трогаются.

## v8
Учредительная версия в parts-модели. Вбирает всю историю base-template до v8: компиляция-паттерн, supersession, иерархия источников, дисциплина (синтез при закрытии единицы работы, расширение схемы только по горизонтали), STATE-механика, роли как опциональная база, граница СТАНДАРТ/КАСТОМ (слоты), лог-дисциплина (дата записи — из окружения, не mtime; lint ловит немонотонность).
